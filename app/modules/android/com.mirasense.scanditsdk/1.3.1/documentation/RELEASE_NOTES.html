Scandit Titanium Plugin - RELEASE NOTES

for iOS and Android version of plugin

Changes in ScanditSDK Plugin for Titanium 1.3.1

  • iOS: fixed bug that was causing camera control issues when the app was accessing the camera both via the Scandit SDK and directly (e.g. to take a picture).

  • Android/iOS: added API method disableStandbyState (under Android invocating this method has no effect, since there is no standby state under Android).

  • Android/iOS: changed Titanium API version in plugin manifest from 3 to 2 for Android and to 1 for iOS. The minSDK version is now set to 2.1.0GA.

  • Android: fixed bug that caused potential app crashes when startScanning or stopScanning were called from a different thread.

  • Android: fixed bug where default scan viewfinder rectangle size was too large.

  • Upgraded to latest version of native Scandit SDK for iOS (3.2.0) and Android (3.5.2)

Changes in ScanditSDK Plugin for Titanium 1.3.0

  • Android: Fixed bug where Scandit API method calls that update the user interface could lead to crashes of the app because of a threading issue.

  • Android: Fixed bug where settings set via the Scandit API were not always executed in the UI when a new activity was created.

  • Upgraded to latest version of native Scandit SDK for iOS (3.1.1) and Android (3.5.1)

Changes in ScanditSDK Plugin for Titanium 1.1.0

Android & iPhone release - February 19th 2013

  • Important software update to work with orientation changes and to support barcode scanning in landscape mode.

IMPORTANT: this update requires a (small) change to the way you instantiate the Scandit SDK in your Titanium app. See the example for details.

  • Fixed a bug in timodule.xml (Android) that prevented deployment on any device. Autofocus and flash are now also optional (no longer required).

  • Fixed a bug where QR code scanning was only enabled by default on iPhone devices and not on Android devices. QR code scanning is now enabled by default on both platforms.

  • Fixed a bug resulting from a typo in the method stopScanning. the method is now called stopScanning (and no longer stopScannning).

  • Corrected toolbar offset.
